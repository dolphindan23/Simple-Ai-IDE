### ✅ Priority 3: Secrets & Keys system (do it right, not “localStorage”)

If you’re installing locally, the clean model is:

#### Secrets should NOT live in:

* localStorage
* plain JSON
* repo files committed to git

#### Best local approach:

* **Encrypted secrets vault file** (on disk), e.g. `~/.simpleide/secrets.enc`
* User sets a master password to unlock vault (or use OS keychain later)

MVP encryption options (choose one):

* **Fast + acceptable:** libsodium / NaCl secretbox
* **Simple + available:** `cryptography` library with Fernet (okay for MVP)
* **Best UX later:** OS keychain (macOS Keychain, Windows Credential Manager, Linux Secret Service)

Then provide:

* Add key / Edit / Delete
* “Reveal” toggle that requires vault unlocked
* “Test connection” button per integration
